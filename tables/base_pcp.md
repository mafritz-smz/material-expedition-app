## Overview

The `BASE PCP` table is the starting point of the expedition process.

Each record represents a Production Order (OB) imported from the production planning process.

This table defines what should be shipped and serves as the parent record for all barcode scans performed during the expedition.

---

# Responsibility

The table is responsible for:

- Defining the Production Order.
- Identifying the expected material.
- Identifying the expected machine.
- Identifying the expected production lot.
- Identifying the expected color.
- Defining the required quantity (Barca).
- Storing the generated label document.

No scan can be performed without selecting a Production Order from this table.

---

# Relationships

```
BASE PCP
     │
     │ One
     ▼
LANÇAMENTOS
```

One Production Order may contain multiple scan records.

Each scan record belongs to one and only one Production Order.

# Virtual Columns

The table also contains calculated information generated by AppSheet.

Examples include:

- Related scan records.
- Current scanned quantity.
- Current scanned weight.
- Remaining quantity.
- Expedition progress.

These values are calculated dynamically and are not physically stored.

---

# Dependencies

The information stored in this table is used by:

- Barcode validation.
- Scan registration.
- Progress calculation.
- PDF generation.
- Label printing.
- Expedition completion.

---

# Business Rules

A Production Order defines every validation performed during the expedition.

Each scanned piece must match the expected:

- Machine
- Lot
- Material
- Color

Only valid scans contribute to the expedition progress.

# Notes

The `BASE PCP` table does not store individual scan events.

Instead, it represents the operational objective of the expedition, while the actual execution is recorded in the `LANÇAMENTOS` table.
